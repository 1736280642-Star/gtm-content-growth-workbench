# JOTO GTM 内容工作台开发任务清单

## 1. 文档定位

本文件基于 `development-plan.md` 拆解，用于进入实际开发排期。

已确认技术路径：

| 类型 | 最终判断 |
|---|---|
| 前端 | React / Next.js + 成熟后台组件库 |
| 后端短期 | Next.js / Node API + Worker 脚本 |
| 后端中期 | 根据复杂度拆出独立 API 服务和任务队列 |
| 数据库 | MySQL |
| 自动化任务 | Node Worker 脚本 |
| 发布能力 | MVP 只做发布队列、导出、URL 回填，不做自动发布 |
| 日志能力 | MVP 先用 Demo CSV / 模拟日志，后续接 Nginx/CDN 日志 |

任务优先级说明：

| 优先级 | 含义 |
|---|---|
| P0 | MVP 必须完成，否则主流程跑不通 |
| P1 | MVP 建议完成，影响可用性和演示完整度 |
| P2 | 后置增强，不影响首版主流程 |

## 11. 建议排期方式

建议不要按页面平均排期，而是按闭环排期：

| Sprint | 目标 | 对应 Phase |
|---|---|---|
| Sprint 0 | 准备资产、搭工程、建数据底座 | Phase 0 + Phase 1 |
| Sprint 1 | 跑通周计划到今日任务 | Phase 2 |
| Sprint 2 | 跑通内容生成到发布队列 | Phase 3 + Phase 4 |
| Sprint 3 | 跑通博客诊断和 GEO 测试 | Phase 5 |
| Sprint 4 | 跑通周报和试运行 | Phase 6 + Phase 7 |

每个 Sprint 必须有一个能演示的结果，不做“底层做了很多但用户看不到”的长周期开发。
