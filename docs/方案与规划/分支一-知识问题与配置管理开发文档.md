# 分支一：知识、问题与配置管理开发文档

## 1. 分支目标

分支：`codex/v5-foundation-knowledge-questions`

本分支建立后续月度生产所依赖的基础能力：

1. `蒸馏词池`升级为`问题与关键词池`。
2. 知识库创建流程收敛为“名称 + 知识库重点 + 导入资料”。
3. 知识库增加关键表达、问题与冲突、证据追溯视图。
4. `AI 配置`更名为`配置管理`，合并现有连接管理。
5. 文章 Prompt 改成表单化`文章表达预设`。
6. 完成导航命名和兼容重定向，为分支二、分支三提供稳定入口。

本分支不实现月度渠道配额、批量正文生成、AI 前台真实采集或官网审计。

## 2. 用户可见结果

```mermaid
flowchart LR
    A["知识库名称与重点"] --> B["导入资料"]
    B --> C["自动识别 Source Chunk Claim"]
    C --> D["关键表达与问题冲突"]
    D --> E["可用知识快照"]

    F["问题与关键词池"] --> G["选择月度目标问题"]
    H["配置管理"] --> I["文章表达预设"]
    H --> J["模型与连接健康"]
```

用户只需要理解：问题、关键词、资料、关键表达、问题与冲突、表达预设。`Source`、`Chunk`、`Claim`只在证据抽屉中作为追溯信息出现，不要求用户维护三套底层表。

## 8. 状态、空状态与错误

| 场景 | 页面表现 | 下一步 |
|---|---|---|
| 没有问题 | 空状态显示`新建第一个目标问题` | 打开问题表单 |
| 问题未冻结 | 可编辑，不能进入正式效果比较 | 冻结或保留探索 |
| 知识库无资料 | 显示知识库重点和`导入资料` | 进入导入页 |
| Source 解析失败 | 显示失败原因，不生成假 Chunk | 重试或重新上传 |
| 冲突阻断生产 | 标明受影响对象和数量 | 进入处理抽屉 |
| 模型缺配置 | 页面仍可浏览，不允许触发对应运行 | 去模型服务补配置 |
| 连接失效 | 显示账号别名和最后成功时间 | 重新连接，不显示凭证 |

## 11. 开发顺序

1. 定义问题、关键词、知识问题和表达预设契约。
2. 完成导航和新路由壳，增加旧路由重定向。
3. 实现问题与关键词池三个页签。
4. 简化知识库创建，补齐详情页签和证据抽屉。
5. 创建配置管理聚合页，迁入模型和连接能力。
6. 实现表单化文章表达预设。
7. 补空状态、错误、权限和审计。
8. 更新结构、交互、路由和浏览器验证。

## 12. 验收标准

1. 用户不再看到`蒸馏词池`和`AI 配置`作为正式名称。
2. 问题与关键词池不出现要求用户理解的“子意图、内容角色”。
3. 已冻结问题修改时生成新版本，不覆盖历史。
4. 创建知识库只要求名称、重点和可选公开范围。
5. 知识库重点不进入 Claim，也不作为文章事实。
6. 问题与冲突至少覆盖十二类问题，并显示影响范围和下一步。
7. 证据抽屉可以从 Claim 追溯 Chunk 和 Source。
