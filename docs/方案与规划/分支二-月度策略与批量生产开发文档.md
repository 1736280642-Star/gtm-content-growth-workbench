# 分支二：月度策略与批量生产开发文档

## 1. 分支目标

分支：`codex/v5-monthly-strategy-production`

本分支把问题、知识和表达预设转换为可批准、可生成、可排程的月度内容：

1. 月度内容矩阵承担`内容策略包`的生成、编辑和批准。
2. 用户按`目标问题 + 文章类型 + 每渠道配额`编排，不展示内容角色或子意图。
3. 多选渠道时，配额表示每个渠道分别生成的数量。
4. `批量生成中心`作为月度内容矩阵子入口，不再是顶层导航。
5. 批量生成中心只执行已批准策略，处理正文生成、质检、异常和排程。
6. 正文详情从任务行打开抽屉或全屏预览，不建立独立导航页面。

本分支依赖分支一已合并后的问题版本、知识快照和文章表达预设契约。本分支不实现 AI 前台采集、月度效果归因或官网审计。

## 2. 核心业务规则

### 2.1 配额语义

```text
渠道成品数 = 每渠道配额 x 所选渠道数
月度渠道成品总数 = 所有配额行的渠道成品数之和
```

示例：

```text
目标问题：企业应该如何选择腾讯云 ADP 服务商？
文章类型：选型与比较
每渠道配额：4
渠道：官网、知乎

结果：官网 4 篇 + 知乎 4 篇 = 8 篇渠道成品
```

若用户希望两个渠道合计 4 篇，应展开渠道设置为官网 2、知乎 2。页面不能把“4 篇”自动平均分配，也不能在多选渠道后隐藏总量增长。

### 2.2 用户可见编排维度

用户只配置目标问题、文章类型、每渠道数量、渠道、产品表达规则包、知识库和文章表达预设。

关键词覆盖、Evidence Preview、去重和证据检查由系统承担。系统内部可以保留 ContentCluster 或 QuestionFacet，但页面不以此要求用户做决策。

## 3. 页面与路由

| 页面 | 正式路由 | 兼容入口 | 本分支职责 |
|---|---|---|---|
| 月度内容矩阵 | `/monthly-matrix` | 保留 | 月份、目标问题、内容策略包、矩阵版本 |
| 内容策略包 | `/monthly-matrix`默认工作区 | 旧策略路径重定向 | 配额编辑、证据预检、批准 |
| 批量生成中心 | `/monthly-matrix/batch-generation` | `/batch-generation`重定向 | 生成任务、人工排程、异常处理 |
| 正文预览 | 查询参数或抽屉 | `/v5/drafts/[id]`不作为导航 | 正文、证据、质检、版本 |
| 当日执行 | `/daily-execution` | 保留 | 暂不改变职责 |
| 数据回传 | `/publish` | 保留 | 暂不改变职责 |

## 5. 月度矩阵任务

策略批准后，一条“ADP 选型 / 选型与比较 / 官网 4 / 知乎 4”配额生成 4 个基础选题建议，每个基础选题展开官网和知乎版本，共 8 个 `ContentMatrixItem`。

同一基础选题的渠道版本共享目标问题和可复用证据候选，但标题、正文、格式、CTA 和发布记录独立。

```text
+--------------------------------------------------------------------------------+
| 矩阵任务 30                          [目标问题 v] [类型 v] [渠道 v] [状态 v]   |
+--------------------------------------------------------------------------------+
| 基础选题 / 渠道版本                    类型       证据状态      状态       操作 |
| 如何评估 ADP 服务商的实施能力                                                 |
|   ├ 官网：企业选择 ADP 服务商应看哪些实施能力  选型比较   [通过]      待批准 [详情]|
|   └ 知乎：选 ADP 服务商时最容易忽略什么       选型比较   [通过]      待批准 [详情]|
| ADP 项目验收需要哪些前置条件                                                   |
|   ├ 官网：...                                  实施指南   [缺证据]    阻断   [缺口]|
|   └ CSDN：...                                  实施指南   [缺证据]    阻断   [缺口]|
+--------------------------------------------------------------------------------+
```

## 7. 状态与约束

策略包状态：

```text
draft -> preview_checking -> pending_review
-> approved / partially_approved -> superseded
```

矩阵任务状态：

```text
draft -> evidence_preview_ready / evidence_gap -> approved
-> ready_for_generation -> generating -> qa_passed / exception -> scheduled
```

强制约束：

1. 配额展开总数不得大于月度渠道成品总数。
2. 提交审核时必须等于月度总数；草稿可以小于总数。
3. 规则包、知识索引和 EvidencePack 必须绑定同一 `sourceSnapshotHash`。
4. 只有批准矩阵项可以进入批量生成。
5. 只有`generatable`或已人工确认降级的任务调用正文模型。
6. 生成时冻结表达预设版本，不读取后来修改的默认值。
7. 渠道版本不可直接复制同一正文，必须经过渠道表达编译和质检。

## 11. 验收标准

1. 页面不要求用户配置内容角色或子意图。
2. “配额 4 + 官网、知乎”明确展开为 8 篇渠道成品。
3. 用户可关闭相同配额并分别设置官网 2、知乎 2。
4. 草稿实时显示月度总数、已分配和待分配；审核前合计必须相等。
5. 策略包保存规则包、知识库、快照和表达预设版本。
6. Evidence Preview 只生成建议和准入状态，不生成正式正文。
7. 批量生成中心不再是顶层导航，旧路径兼容重定向。
8. 生成页不能修改月度策略字段。
9. 批量生成跳过阻断任务，并给出逐项原因和下一步。
10. 正文从任务行预览，导航中不存在正文详情入口。
11. 事实句可从正文追溯到 Claim、Chunk 和 Source。
12. 桌面端线框中的配额、任务状态和操作入口无文字重叠。
13. `npm.cmd run typecheck`、`npm.cmd run validate:structure`、相关 smoke 和生产构建通过。
