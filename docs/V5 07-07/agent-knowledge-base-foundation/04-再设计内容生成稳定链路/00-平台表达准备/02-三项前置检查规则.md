# 平台表达三项前置检查规则

版本：v1.1.0

## 适用范围

用于公众号平台表达准备，也作为知乎、今日头条、掘金和 CSDN 改写前的共同事实边界。检查对象不是单独一个标题，而是已经选定的内容类型、标题类别、受众、证据依据和标题候选。

## 必填输入

- `product`
- `channel`
- `contentType`
- `platformContentType`
- `platformExpressionProfileId`
- `titleCategory`
- `targetAudience`
- `sourceProblem`
- `productExpressionRulePackageId`
- `evidenceBasis`
- `title`
- `brandMotherSentenceUsed`

缺少产品表达规则包、平台内容类型、来源问题或证据依据时，可以生成待审草稿，但不得人工确认或批量进入正文生成。

## 检查一：事实和数字是否有证据

检查标题和内容结构中的百分比、数量、规模、时长、速度，第一、唯一、最强、最快、重新定义，客户结果、行业排名和市场总体判断。

通过条件：

- 能指向官网、已授权案例、产品文档、FAQ 或可靠调研；
- `EvidencePreview` 的可用 Claim、禁用 Claim 和证据缺口与标题一致；
- 匿名代表性案例不写成可识别客户事实；
- 内容类型要求的数据、体验或趋势判断确实具备对应材料。

不通过时删除数字、缩小表达，或标记 `evidence_missing` 并禁止确认。

## 检查二：内容结构与标题承诺能否被正文证明

通过条件：

- 第一屏按所选内容类型进入真实场景、行业变化或具体任务；
- 主体结构符合该内容类型的推荐逻辑，而不是只替换小标题；
- 正文解释标题中的关键因果；
- 产品植入强度与内容类型一致；
- 标题写 N 个标准时，正文存在对应 N 项；
- 结尾动作符合显性推广或隐性种草的转化强度。

不通过时调整内容类型、缩小标题或补充正文证据，并标记 `body_unprovable`。

## 检查三：是否越过产品能力与人机角色边界

高风险表达包括：只靠聊天管好网络、AI 自动替企业做决定、完全替代专业岗位、无需审批修改配置、一键解决、零风险、100% 安全。

通过条件：

- AI 被写成入口、辅助、取证、整理、检测、建议或流程承载；
- 专业判断、审批、复核、回滚和责任仍归人；
- 标题、结构和 CTA 均与产品表达规则包边界一致；
- 不把规划能力、演示能力或目标状态写成已稳定交付结果。

不通过时标记 `role_boundary_risk`，必须人工修改。

## 人工确认门槛

系统输出：

- `evidenceSupported`
- `bodyProvable`
- `roleBoundarySafe`
- `notes`
- `evidencePreviewId`
- `checkedAt`
- `checkRuleVersion`

三个布尔结果必须全部为 `true`，且人工确认所选 `platformContentType`、`titleCategory` 和 `targetAudience` 后，任务才可被冻结并请求 Final `EvidencePack`。Final `EvidencePack` 仍可能因标题级证据不足、来源失效或规则版本变化而拒绝生成。旧任务缺少检查结果时显示“待补平台表达准备”，不能静默视为通过。
