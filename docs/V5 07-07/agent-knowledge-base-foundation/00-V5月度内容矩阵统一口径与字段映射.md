# V5 月度内容矩阵统一口径与字段映射

## 1. 文档定位

本文件是 V5 07-07 全部阶段文档的周期、流程和字段总约束。

月度口径以 D:/GTM/工作台/docs/V5 -07-09 的三份设计为准。阶段一至阶段六的领域设计必须服从本文件；当旧文档中出现周计划、今日发布生成、逐篇人工审核、人工发布主链路或周报主复盘时，按本文件解释和迁移。

## 2. 唯一正式主流程

    产品与知识库治理
    → 产品表达规则包确认
    → 产品进入月度内容生产池
    → 月度目标输入
    → Workflow Agent 生成月度内容策略包草稿
    → 规则引擎校验策略包
    → 人工审核月度策略
    → 生成月度内容矩阵草稿
    → 矩阵项 EvidencePreview 与平台表达准备
    → 人工审核月度矩阵
    → 为已批准矩阵项生成 Final EvidencePack
    → Final Evidence Gate
    → 批量生成一个月正文
    → 硬规则与评测 Runner 自动质检
    → 合格稿与异常稿自动分流
    → 异常稿人工处理，合格稿抽检
    → 进入定时发布排程
    → 人工发布前置确认
    → 定时自动化发布或人工接管
    → URL 与渠道数据回传
    → 月度复盘
    → 下月策略优化

## 3. 主入口与派生视图

| 对象或页面 | V5 正式职责 | 是否业务真源 |
|---|---|---|
| Monthly Content Matrix | 月度目标、策略包、矩阵审核和批量生产主入口 | 是 |
| Weekly View | 从已批准月度矩阵按周过滤后的执行视图 | 否 |
| Today Execution | 查看当天排程、发布状态、失败接管和 URL 确认 | 否 |
| Batch Generation Center | 按已批准矩阵批量生成和自动质检 | 是，运行真源 |
| Draft Sampling and Exception | 处理异常稿并抽检合格稿 | 是，处理记录真源 |
| Publish Schedule | 定时发布、失败重试和人工接管 | 是 |
| Monthly Review | 主要复盘与下月策略输入 | 是 |
| Weekly Snapshot | 月内过程监控快照 | 否 |

Weekly View 和 Today Execution 不得创建脱离 MonthlyPlan 的独立任务，也不得重新决定产品、渠道、标题、主蒸馏词、规则包或发布日期。

## 4. 人工判断位置

人工判断从逐篇从零审核前移到以下位置：

1. 产品表达规则包激活。
2. 月度策略包审核。
3. 月度内容矩阵审核，可批准可生成部分并把缺口项送入异常队列。
4. 高风险或异常稿处理。
5. 合格稿抽检。
6. 定时正式发布前置确认。
7. 版本激活、回滚和风险例外。

普通合格稿不要求逐篇人工全文审核，但必须通过自动质检并进入抽检策略。

以下字段名是 V5 新写入和跨阶段接口的唯一正式口径：

| 正式字段 | 含义 | 禁止继续作为新字段使用的旧别名 |

### 5.9 EvidencePreview 与 Final Evidence Gate

两个证据节点不能混为一个状态：

1. `EvidencePreview` 在月度策略批准、矩阵草稿生成后运行，用于矩阵审核、标题与平台表达准备；它不授予正文生成权限。
2. 月度策略包使用自身的 `evidenceReadinessSummary` 支撑策略审核，不提前创建 ContentMatrixItem。
3. 月度矩阵人工批准后，为每个批准项生成 Final `EvidencePack`。
4. `evidenceGateStatus` 只能由 Final `EvidencePack` 决策写入；通过后才可把矩阵项加入 `BatchGenerationRun`。

正式 `evidenceGateStatus` 使用：`generatable`、`generatable_with_downgrade`、`needs_material`、`needs_review`、`blocked`、`pending_config`。其中只有 `generatable` 和完成明确人工降级确认后的 `generatable_with_downgrade` 可以进入批量生成。

### 月度完成条件

MonthlyPlan 只有在以下条件满足后才能 completed：

1. 所有必需矩阵项已经 published、cancelled 或有授权的未完成解释。
2. URL 和渠道结果真实回传。
3. 发布失败项已经关闭或转入后续处理。
4. 月度复盘已经生成并经负责人确认。

## 8. Agent 与发布自动化边界

Agent 可以：

- 根据月度目标和已激活治理资产生成策略包草稿。
- 在规则约束下生成矩阵草稿并拆成周视图和日任务。
- 对已批准矩阵生成 Final EvidencePack、执行 Final Evidence Gate、批量生成、自动质检和异常分流。
- 创建、暂停、恢复已批准的 PublishSchedule。
- 解释发布失败并执行安全重试或路由人工接管。
- 生成月度复盘草稿和下月建议。

Agent 不可以：

- 自行批准月度策略或月度矩阵。
- 把 evidence_gap 矩阵项强行标记为可生成。
- 绕过自动质检和发布前置确认。
- 临时修改已批准矩阵的产品、渠道、标题、主蒸馏词或发布时间。
- 在平台未配置时伪造已发布状态和 URL。
- 发起不属于已批准 PublishSchedule 的即时自主发布。

定时自动化发布由发布执行服务按已批准排程执行，不等于 Agent 拥有自由发布权。

## 9. 兼容原则

- V4 weekly-plan 页面保留，但降级为 MonthlyPlan 的周视图。
- V4 today 页面保留，但降级为当天执行台。
- V4 drafts 页面转为异常稿处理与合格稿抽检。
- V4 publish 页面升级为 PublishSchedule。
- V4 weekly report 保留为 WeeklySnapshot，V5 主复盘新增 MonthlyReview。
- 兼容字段只用于迁移和读取，不允许新 V5 写入继续以周计划为业务真源。

## 10. 验收标准

1. 任一生成任务都能追溯到 monthlyPlanId、matrixVersionId 和 matrixItemId。
2. Weekly View 中的调整会更新月度矩阵版本，不产生独立周计划真源。
3. Today Execution 不能发起一个月正文批量生成。
4. 月度矩阵未经人工批准不能生成 Final EvidencePack 或创建 BatchGenerationRun。
5. 每个进入 BatchGenerationRun 的矩阵项都有 Final EvidencePack，且 Final Evidence Gate 已通过。
6. 合格稿进入 PublishSchedule，异常稿进入人工异常队列。
7. 正式自动发布必须有人工前置确认和真实平台能力。
8. 周报只作为过程快照，不能替代 MonthlyReview。
9. 旧字段迁移后保留来源引用，不静默丢失历史。
10. 缺配置、发布失败和数据缺失都显示真实状态。
