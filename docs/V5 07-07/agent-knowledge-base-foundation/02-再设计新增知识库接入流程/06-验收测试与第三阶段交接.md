# 验收测试与第三阶段交接

## 1. 目标

定义第二阶段如何证明“新增资料已经成为可信治理资产”，以及第三阶段 RAG 可以读取哪些数据、不能自行补哪些判断。

验收不能只看上传成功、解析成功或生成了规则草稿。三者都不等于资料可以进入生产检索。

## 2. 端到端验收路径

```text
真实资料导入
-> 来源分类
-> 产品归属
-> 原子事实
-> 证据与冲突
-> 规则包差异
-> 人工确认
-> 生效版本
-> RAG 准入清单
```

每个环节都必须保留输入、输出、状态、责任人和失败原因。

## 3. 固定验收样本

### 3.1 通用样本

| 编号 | 输入 | 预期 |
| --- | --- | --- |
| S01 | 当前官网产品页 | 识别为 A2，抽取产品身份和公开能力 |
| S02 | 官方博客 | 默认 B2，不单独升级性能或产品强事实 |
| S03 | 隐私政策 | 进入隐私/数据流事实，不混入普通能力 |
| S04 | 性能营销数字但无测试条件 | 生成候选事实和阻断缺口，不生成强表达 |
| S05 | 经授权案例 | 保留客户、范围、基线、时间和授权状态 |
| S06 | 匿名案例 | 标记代表性作用域，不泛化 |
| S07 | 规划中功能 | 状态为 planned，不进入当前能力 |
| S08 | 同 URL 新版本 | 创建来源修订，保留旧版本和差异 |
| S09 | 完全重复资料 | 不创建重复生产来源 |
| S10 | 跨产品文章 | 拆分产品事实组，不错误归属 |
| S11 | 登录/拦截/空页面 | 解析失败或隔离，不抽取事实 |
| S12 | 包含密钥或客户隐私 | G0 阻断，不发送外部模型 |

### 3.2 Pharaoh Command

1. “自动执行”被拆成工作流能力、写权限和人工审批条件。
2. 多厂商支持在无兼容清单时只生成条件表达。
3. 匿名案例数字保留 `representative_anonymized_case`。
4. 数据不出网承诺在无部署架构时被阻断。

### 3.3 Noteflow

1. 官网和隐私政策对第三方处理器的差异形成冲突组。
2. 当前功能、规划功能和历史博客主张分开。
3. 私有部署和公有服务的数据边界分别记录。
4. 解析能力数字在缺评测条件时被降级。

### 3.4 唯客 AI 护栏

1. PII 的 5 类、10+、20+ 形成版本/页面冲突组。
2. `<300ms` 缺测试环境、分位值和策略规模时不能生效。
3. “帮助合规”不会转成“保证合规”。
4. Dify 集成范围按应用类型和版本记录。

## 4. 量化指标

### 4.1 来源层

- 资料类型准确率：`>= 95%`。
- 高风险资料人工复核覆盖率：`100%`。
- 错误页、空页、敏感资料进入事实抽取：`0`。
- 完全重复生产来源新增率：`0`。

### 4.2 实体与事实层

- 产品归属准确率：`100%`；不确定项进入人工确认。
- 强事实原文定位完整率：`100%`。
- 数字限定条件保留率：`100%`。
- planned/beta 误判为 current：`0`。
- 外部行业事实误归为产品能力：`0`。

### 4.3 规则与治理层

- 生效变更可追溯率：`100%`。
- 新资料静默覆盖 active 版本：`0`。
- 阻断冲突未解决却激活规则包：`0`。
- 高风险变更责任角色确认覆盖率：`100%`。
- 回滚影响对象可识别率：`100%`。

### 4.4 候选层

- 知识库生成蒸馏词具有 `sourceClaimIds`：`100%`。
- 用户问题建议能说明可回答状态：`100%`。
- 无资料支撑却标记为自动验证通过：`0`。

## 5. 业务验收问题

知识库维护人员应能回答：

1. 这份资料是什么类型，为什么能或不能证明产品事实？
2. 它属于哪个产品，是否存在相似实体或跨产品风险？
3. 系统抽取了哪些事实，每条事实原文在哪里？
4. 新资料改变了哪些规则，哪些只是重复确认？
5. 哪些内容缺证据，应该找谁补什么资料？
6. 哪些冲突必须裁决，当前采用什么临时口径？
7. 哪些蒸馏词和问题可以进入内容规划？
8. 当前哪些资料允许第三阶段检索？

如果这些问题只能靠查看原始模型回答或开发日志才能回答，第二阶段不算完成。

## 6. 第三阶段准入契约

第三阶段只能默认读取满足以下条件的来源和事实：

### 6.1 SourceAsset 准入

- `status = approved_for_claim_extraction` 或后续正式生产状态。
- `visibility = public`；内部任务可按权限读取 `internal`。
- 不处于 `isolated / deprecated / parse_failed`。
- 已确认 `productId`、`documentType`、`authorityLevel`。
- 有稳定 `sourceRevisionId`、正文和原文定位能力。

### 6.2 ProductClaim 准入

- `reviewStatus = supported | conditional`。
- 有 `sourceId + sourceRevisionId + sourceLocator`。
- 产品、能力状态、适用范围和版本可用。
- 不属于未解决阻断冲突。
- `planned / beta / deprecated` 有明确状态，检索路由不得当作 current。

### 6.3 RulePackage 准入

- 正式内容生产只读取 `active` 版本。
- 草稿可以进入治理预览和评测沙箱，不进入生产 EvidencePack。
- 第三阶段不得自行改变允许、条件和禁止表达。

## 7. 第二阶段向第三阶段输出的数据

```yaml
ragIngestionManifest:
  productId: ""
  knowledgeBaseIds: []
  activeRulePackageVersionId: ""
  approvedSourceRevisionIds: []
  approvedClaimIds: []
  blockedClaimIds: []
  unresolvedConflictIds: []
  authorityPolicyVersion: ""
  generatedAt: ""
```

第三阶段以该 Manifest 作为生产切片输入边界，避免直接扫描知识库里所有已上传文本。

## 8. 第三阶段不得反向修改的字段

RAG 切片、召回和 EvidencePack 可以引用，但不能修改：

- 产品实体归属。
- `documentType / authorityLevel / visibility`。
- 产品事实和能力状态。
- 来源冲突裁决。
- 规则包允许、条件和禁止表达。
- 审批记录。

第三阶段发现问题时只能创建：

- 新的资料缺口建议。
- 来源质量问题。
- 事实冲突候选。
- Chunk 或检索 Badcase。

这些建议回到第二阶段治理流程确认。

## 9. 失败与恢复测试

必须测试：

1. 抓取 Provider 未配置时返回 `pending_config`。
2. 批次部分资料解析失败，成功资料仍可审核。
3. 模型抽取失败可重试，不重复创建事实。
4. 人工修正资料类型后可以重跑事实抽取并保留旧记录。
5. 生效前来源被隔离时，规则包激活被服务端阻断。
6. 激活过程中版本已变化时返回 `version_conflict`。
7. 回滚后第三阶段 Manifest 和缓存被标记刷新。

## 10. 上线策略

1. 先对三个现有产品各选 10–20 份代表资料做沙箱验收。
2. 先验证来源分类、事实和冲突，不立即重切全部历史博客。
3. 规则包 V5 与现有 V4 草稿双写一段时间。
4. 三产品验收通过后，第三阶段只消费 V5 准入 Manifest。
5. 最后再批量处理 Noteflow 和唯客 AI 护栏的大量历史文章。

## 11. 月度内容矩阵交接与阶段完成标准

### 11.1 月度内容矩阵交接要求

第三阶段接收的不只是可检索来源，还必须接收：

- productId 与 activeRulePackageVersionId。
- MonthlyProductionReadiness 状态。
- allowedContentTypes、conditionalContentTypes 和 blockedContentTypes。
- allowedChannels 与 requiredEvidenceRoles。
- evidenceGapIds 与 maxMonthlyQuota。
- 蒸馏词的本月矩阵覆盖候选。

月度策略审核使用 MonthlyStrategyPackageVersion 自身的 `evidenceReadinessSummary`。策略批准并生成矩阵草稿后，第三阶段为矩阵审核生成 EvidencePreview；矩阵批准后，再为 ContentMatrixItem 生成 Final EvidencePack 并写入正式 Evidence Gate。它不能把未进入月度生产池的产品判为可生成。

### 11.2 阶段交付完成标准

- 设计文档、数据契约、状态和权限口径统一。
- 三产品固定样本具备预期答案。
- 第二阶段输出可直接成为第三阶段输入。
- 未确认资料无法绕过准入进入生产检索。
- 所有风险都能翻译成补资料、人工确认、降级或阻断动作。
- 月度生产准备度、矩阵适用范围和生产池状态可被第三阶段直接消费。
