## 2. 发现的冲突与不妥

### 2.1 策略审核与矩阵生成顺序冲突

部分文档把 EvidencePreview、平台表达准备或矩阵生成写在月度策略人工批准之前，导致策略对象和矩阵对象互相依赖。

修正：月度策略审核只读取 `evidenceReadinessSummary`；策略批准后才生成矩阵草稿和矩阵项 EvidencePreview。

### 2.2 Evidence Gate 存在循环定义

部分文档同时把 Evidence Gate 写在矩阵批准前，又把 Final EvidencePack 写在矩阵批准后，任务图还出现“先建批次、后建 Final Pack”的倒序。

修正：拆成两个明确节点。

- EvidencePreview：矩阵审核前，仅支持标题、平台表达和风险预览。
- Final Evidence Gate：矩阵批准后，由 Final EvidencePack 决策；通过后才能加入 BatchGenerationRun。

### 2.3 跨阶段字段存在多套命名

RAG、生成和评测文档混用了 `taskType / contentType`、`targetChannel / channel`、`topic / platformTitle / title`、`sourceQuestion / sourceProblem`、`rulePackageVersion / rulePackageVersionId`、`DraftRevision / DraftVersion / articleDraftVersion`。

修正：V5 新写入统一使用：

- `contentType`
- `channel`
- `title`
- `sourceProblem`
- `productId`
- `rulePackageVersionId`
- `draftVersionId`

旧字段只允许迁移读取，不再进入新契约。

### 2.4 发布能力与发布状态混淆

原状态表没有为 `draft_only` 和 `manual_only` 定义独立结果，存在被误计为正式发布的空间。

修正：

- `formal_publish` 真实成功后才写 `published` 和公开 URL。
- `draft_only` 写 `draft_created`。
- `manual_only` 写 `manual_handoff_ready`。
- 缺配置写 `pending_config`。

MonthlyReview 不把后三者计为正式发布成功。

### 2.5 核心实体定义不完整

统一口径已有 MonthlyPlan、ContentMatrixItem、BatchGenerationRun 和 PublishSchedule，但缺少 ContentMatrixVersion 与 MonthlyReview 的正式字段定义。

修正：补齐两个实体、版本状态、真实指标和审批字段。

### 2.6 `V5 -07-09` 引用的源方案缺失

`V5 -07-09/01-重构后影响范围分析.md` 引用了 `workflow-agent-content-production-system/workflow-agent方案.md`，但文件不存在。

修正：恢复总方案入口，并明确详细契约只以 `agent-knowledge-base-foundation` 为真源，避免再建一套平行设计。

### 2.7 JOTO 品牌缺少规则包边界

阶段一只包含三个自研产品规则包，缺少 `V5 -07-09` 月度生产池和 JOTO 官方内容所需的品牌规则边界。

修正：新增 JOTO 品牌规则边界草案，覆盖 Dify 合作身份、最佳服务商关键词、服务范围、客户案例、自研产品归属和月度准入。

### 2.8 三个现有产品没有明确月度准入状态

产品规则文档虽然有表达边界，但没有明确当前草案不能自动进入月度生产池。

修正：Pharaoh Command、Noteflow、唯客 AI 护栏均补充 `monthlyProductionReady=false`、可评估内容类型、条件内容和渠道范围。

### 2.9 平台表达存在独立真源风险

TopicMasterBrief、PlatformExpressionBrief 和人工确认字段可能形成一套脱离 MonthlyPlan / ContentMatrixItem 的计划与审批链。

修正：主题母版只作为策略包内的同主题分组；每个平台版本必须落成独立 ContentMatrixItem；最终确认统一归入月度矩阵审批。

### 2.10 知识库快照容易被误解为 V5 已准入资产

原始快照包含 `enabled`、`real_embedding`、测试博文、备份页和 404 页面。这些状态只能证明 2026-07-10 的 V4 系统现状，不能证明 V5 SourceAsset、ProductClaim、active 规则包或 MonthlyProductionReadiness 已通过。

修正：保留原始快照不篡改，在资产 README 增加迁移边界和重新治理要求。

### 2.11 AI 配置 / 治理日志入口不够明确

阶段六有底层日志设计，但总索引未显式覆盖 `V5 -07-09` 要求的 AI 配置 / 治理页面。

修正：补充 Provider、Prompt、RAG、Rule Engine、Eval Runner、Agent Runs 和 Publish 日志页面职责及业务数据边界。
