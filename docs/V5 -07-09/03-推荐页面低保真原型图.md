# V5 推荐页面低保真原型图：月度内容矩阵版

## 设计判断

V5 页面不应该围绕“今天写哪篇”设计，而应该围绕“这个月的内容矩阵是否可稳定生产并按时发布”设计。

页面设计目标：

1. 内容增长负责人看到的是：本月目标、产品配额、渠道分布、矩阵风险和下月优化。
2. 工作台运营者看到的是：哪些产品已具备批量生产条件，哪些异常需要处理。
3. 内容发布人员看到的是：哪些稿件已通过、哪些需要抽检、哪些发布失败需要接管。
4. 知识库维护者看到的是：哪些资料缺口正在阻塞月度矩阵。
5. 开发管理员看到的是：Provider、RAG、规则引擎、评测 Runner、发布排程和 Agent 调度日志。

签名元素建议：月度内容矩阵上的 “Evidence Gate”。每个矩阵项在批量生成前都必须显示可生成、缺证据、规则风险或待配置。

## 8. 新增页面：定时发布排程

页面职责：管理一个月内合格稿的自动化发布时间和发布状态。

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 定时发布排程  2026-07                         [同步平台状态] [暂停全部]     │
├────────────────────────────────────────────────────────────────────────────┤
│ 发布日历                                                                    │
│ ┌──────────────────────────────────────────────────────────────────────┐   │
│ │ 日期   时间   标题       平台    状态          URL / 失败原因          │   │
│ │ 07-10 10:00  ...        CSDN    已发布        https://...             │   │
│ │ 07-10 14:00  ...        知乎    待发布        -                       │   │
│ │ 07-11 10:00  ...        掘金    发布失败      auth_required           │   │
│ └──────────────────────────────────────────────────────────────────────┘   │
│                                                                            │
│ 失败处理抽屉                                                                │
│ - 失败原因：平台登录态失效                                                   │
│ - 下一步：[重试] [人工接管] [暂停该平台]                                     │
└────────────────────────────────────────────────────────────────────────────┘
```

真实状态规则：

1. 没有真实平台配置时显示 `pending_config`。
2. 只支持草稿的平台显示“草稿已创建”，不显示“已发布”。
3. 正式自动发布必须记录发布时间、平台、返回 URL 或失败原因。

## 推荐页面新增顺序

第一批：

1. 月度内容矩阵。
2. 产品表达规则包月度准入扩展。
3. 批量生成中心。
4. 定时发布排程。

第二批：

1. Agent 底座总览。
2. 异常队列。
3. 月度复盘。
4. AI 配置页 Agent / Publish 日志。

第三批：

1. Workflow Agent 自动生成月度策略包。
2. Agent 自动拆解矩阵。
3. Agent 自动分流和下月复盘建议。
