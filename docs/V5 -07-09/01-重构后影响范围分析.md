# V5 月度内容矩阵与 Agent 知识库底座重构影响范围分析

## 来源校正

已找到方案来源：`D:/GTM/工作台/docs/V5 07-07/workflow-agent-content-production-system/workflow-agent方案.md`。

该方案明确写到：

1. 从“周计划 + 正文生成 + 草稿质检”升级为“月度策略审核 + Workflow Agent 批量生产 + 规则引擎拦截 + 异常队列处理”。
2. 支持多产品、多渠道、月度级内容生产。
3. 人工从逐篇正文审核，前移为月度策略审核和异常处理。
4. 整体链路是：月度目标输入 -> 月度内容策略包 -> 人一次性审核月度策略 -> 拆成周计划 / 日任务 -> 批量生成正文 -> 自动分流。
5. 新增产品确认规则包后进入月度内容生产池。

因此，V5 的主流程应改为“月计划 / 月度内容矩阵”，不是以周计划为主。

## 核心结论

V5 不应该继续围绕“周计划”做主入口，而应新增或重构为“月度内容矩阵”主入口。

推荐新主链路：

```text
Agent 知识库底座
-> 产品规则包确认
-> 月度目标输入
-> 月度内容策略包
-> 月度内容矩阵
-> 人工审核月度矩阵
-> 批量生成一个月正文
-> 自动质检与异常分流
-> 人工处理异常稿
-> 合格稿进入定时发布排程
-> 定时自动化发布
-> URL / 数据回传
-> 月度复盘
-> 下月策略优化
```

现有 V4 的 `weekly-plan`、`today`、`drafts`、`publish` 不应直接删除，而应降级为执行视图：

1. `weekly-plan`：从主计划入口降级为月度矩阵的周视图。
2. `today`：从正文生成主入口降级为当日发布执行台。
3. `drafts`：从逐篇审核页升级为异常稿处理页和抽检页。
4. `publish`：从人工发布队列升级为定时发布排程与自动化发布状态页。

## 会影响哪些页面

### 1. 首页 / Dashboard

影响方式：从“本周执行概览”升级为“本月内容生产与发布概览”。

需要新增：

1. 本月内容矩阵总量。
2. 本月已生成正文数。
3. 已通过自动质检数。
4. 异常稿待处理数。
5. 已进入定时发布数。
6. 已自动发布数。
7. 自动发布失败数。

保留：

1. 今日待处理任务。
2. URL 回填状态。
3. 数据回传状态。

### 2. 工作台设置 / Settings

影响方式：从“默认产品/品牌计划”升级为“产品月度生产配置”。

需要预留：

1. 产品实体管理。
2. 默认知识库多选。
3. 默认产品表达规则包。
4. 默认月度配额。
5. 默认渠道比例。
6. 默认发布频率。
7. 默认自动发布策略。

对当前 `WorkspaceSetting.productPlans` 的影响：

1. 保留现有周配额字段做兼容。
2. 新增月度配额和月度渠道分布字段。
3. 后续从固定 `ProductKey` 逐步升级为产品实体。

### 3. 知识库 / Knowledge

影响方式：知识库成为月度内容矩阵的前置准入条件。

需要新增：

1. 资料权威等级。
2. 关联产品。
3. 规则包状态。
4. chunk 元数据完整度。
5. EvidencePack 可用性。
6. 月度生产可用状态。

判断规则：

1. 没有 active 规则包的产品不能进入月度内容生产池。
2. 缺官网证据的产品不能生成强转化类内容。
3. 缺案例证据的产品不能生成案例 / ROI / 对比类内容。

### 4. 知识库导入 / Knowledge Import

影响方式：导入资料后不只是入库，还要决定该资料是否能支撑月度矩阵。

需要新增步骤：

```text
选择产品
-> 选择资料类型
-> 标记权威等级
-> 解析与切片
-> 生成规则包草稿
-> 生成蒸馏词 / 用户问题
-> 检查月度内容可生产范围
```

### 5. 产品表达规则包页

影响方式：这是月度批量生成前的硬准入页面。

需要新增：

1. 产品身份与定位。
2. 允许表达。
3. 禁止表达。
4. 证据要求。
5. 渠道表达边界。
6. 月度矩阵适用范围。
7. 版本确认与回滚。

关键规则：

1. Agent 可以生成规则包草稿。
2. Agent 不能让规则包自动生效。
3. 月度矩阵只能使用 active 规则包。

### 6. 蒸馏词池 / Distilled Terms

影响方式：蒸馏词不只是词池，而是月度内容矩阵的主题种子。

需要新增：

1. 月度矩阵覆盖状态。
2. 对应产品。
3. 对应内容类型。
4. 对应渠道。
5. 是否已进入本月矩阵。
6. 是否来自 GEO 缺口 / 知识库 / 搜索问题 / Badcase。

### 7. 新增主页面：月度内容矩阵 / Monthly Content Matrix

影响方式：替代“周计划”成为 V5 主计划入口。

页面职责：

1. 输入月度目标。
2. 生成月度内容策略包。
3. 按产品、渠道、内容类型生成一个月内容矩阵。
4. 人工一次性审核矩阵。
5. 确认后进入批量生成。

需要展示：

1. 产品配额。
2. 渠道配比。
3. 内容类型覆盖。
4. 蒸馏词覆盖。
5. 证据准备度。
6. 发布日历。
7. 风险和缺口。

### 8. 周视图 / Weekly View

影响方式：保留为月度矩阵的周拆解视图，不再作为主入口。

职责：

1. 查看某一周从月度矩阵拆下来的任务。
2. 做轻量调整。
3. 不重新决定产品、渠道、主蒸馏词和规则包。

### 9. 当日执行 / Today

影响方式：从“批量生成正文”主入口降级为当日发布执行台。

职责：

1. 查看今天排程的内容。
2. 查看是否已生成、已质检、已入发布队列。
3. 处理自动发布失败。
4. 回填或确认 URL。

不再承担：

1. 月度批量生成主入口。
2. 重新决定标题、渠道、产品和主蒸馏词。

### 10. 批量生成中心 / Batch Generation

影响方式：新增或从 Today 中拆出。

职责：

1. 按月度矩阵批量生成正文。
2. 为每篇文章生成 EvidencePack。
3. 调用规则引擎硬校验。
4. 调用评测 Runner 软评分。
5. 自动分流合格稿和异常稿。

### 11. 草稿预览 / Drafts

影响方式：从逐篇人工审核转为“异常稿处理 + 抽检”。

需要新增：

1. 异常原因。
2. 规则引擎结果。
3. 评测 Runner 结果。
4. Badcase 归因。
5. 处理动作：补知识库、补规则包、局部改写、保留原因。

普通合格稿不应要求逐篇人工审核，只进入抽检或发布排程。

### 12. 发布 / Publish

影响方式：从平台草稿队列升级为“定时自动化发布排程”。

需要新增：

1. 发布日历。
2. 定时发布时间。
3. 平台账号 / 渠道。
4. 自动发布状态。
5. 失败原因。
6. 重试状态。
7. 人工接管入口。

真实边界：

1. 如果平台 API / bridge 未配置，显示 `pending_config`。
2. 如果平台只支持草稿，不支持正式发布，状态必须显示为“仅草稿自动化”。
3. 定时正式自动发布必须有人工审核通过的前置状态。

### 13. 周报 / 月报

影响方式：V5 主复盘应升级为月度复盘，周报保留为过程快照。

月度复盘需要新增：

1. 月度矩阵完成率。
2. 自动生成成功率。
3. 自动质检通过率。
4. 异常稿原因分布。
5. 定时发布成功率。
6. 渠道数据回传率。
7. GEO / AI 可见性变化。
8. 下月策略建议。

### 14. AI 配置 / AI Governance

影响方式：成为月度批量生产的底层治理日志中心。

需要新增：

1. 月度策略包生成记录。
2. Agent run 记录。
3. RAG 检索路由日志。
4. EvidencePack 生成记录。
5. 规则引擎校验记录。
6. 评测 Runner 记录。
7. 自动发布任务日志。

## 会影响哪些接口

### 建议新增接口

1. `/api/monthly-plans`
   - 创建、读取、更新月度内容计划。

2. `/api/monthly-plans/generate`
   - 根据月度目标、产品配额、规则包和蒸馏词生成月度内容策略包。

3. `/api/content-matrix`
   - 管理月度内容矩阵。

4. `/api/content-matrix/[id]/approve`
   - 人工审核通过月度矩阵。

5. `/api/batch-generation-runs`
   - 按矩阵批量生成正文并记录运行批次。

6. `/api/evidence-packs`
   - 生成和查看 EvidencePack。

7. `/api/rule-engine/validate`
   - 校验任务、证据包、草稿和发布排程。

8. `/api/evaluation-runs`
   - 评测 Runner 执行记录。

9. `/api/publish-schedules`
   - 创建、查看、暂停、恢复定时发布任务。

10. `/api/publish-schedules/[id]/run`
    - 触发单条定时发布执行。

11. `/api/agent-runs`
    - Workflow Agent 月度策略、拆解、分流和复盘记录。

### 需要扩展的现有接口

1. `/api/knowledge-bases`
   - 增加产品归属、资料权威等级、月度生产可用状态。

2. `/api/knowledge-bases/[id]/product-expression`
   - 扩展规则包字段和月度矩阵适用范围。

3. `/api/distilled-terms`
   - 增加月度矩阵覆盖状态。

4. `/api/article-drafts`
   - 支持批次生成来源和矩阵项来源。

5. `/api/distribution-targets`
   - 支持定时发布排程状态。

6. `/api/ai-governance`
   - 聚合月度策略、批量生成、自动质检和发布排程日志。

## 对当前流程的核心变化

当前 V4 流程：

```text
周计划
-> 今日发布
-> 草稿质检
-> 人工发布
-> URL 回填
-> 数据回传
-> 周报复盘
```

V5 校正后流程：

```text
产品 / 知识库治理
-> 规则包确认
-> 月度目标输入
-> 月度策略包
-> 月度内容矩阵
-> 人工审核月度矩阵
-> 批量生成一个月正文
-> 自动质检与异常分流
-> 异常稿人工处理
-> 合格稿进入定时发布排程
-> 定时自动化发布
-> URL / 数据回传
-> 月度复盘
-> 下月策略优化
```

## 最小改造优先级

第一优先级：

1. 新增月度内容矩阵文档与页面入口。
2. 产品规则包扩展为月度矩阵准入条件。
3. EvidencePack 支持矩阵项级生成。
4. 批量生成中心支持按月度矩阵执行。

第二优先级：

1. 异常队列。
2. 定时发布排程。
3. 月度复盘。

第三优先级：

1. Workflow Agent 自动生成月度策略包。
2. Agent 自动拆解矩阵。
3. Agent 自动分流和复盘建议。
