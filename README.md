#  基于GEO优化的自动化文章发布的工作台

 基于GEO优化的自动化文章发布的工作台，用来支持产品与品牌的选题规划、正文生成、草稿质检、人工发布、数据回传、GEO 诊断和周度复盘。

V3 版本不再把它定位成一个通用 AI 写作 Demo，而是围绕真实内容增长流程做收敛：每个页面只承担自己该负责的环节，让内容从计划、生产、发布、回传到复盘形成闭环。

```text
周计划 -> 今日发布 -> 草稿质检 -> 人工发布 -> URL 回填 -> 数据回传 -> 诊断分析 -> 周报复盘 -> 下周计划
```

## 项目定位

这个工作台同时服务两个目标：

1. 短期目标：支持 JOTO 内容推广、发布执行和线索获取。
2. 长期目标：把重复出现的内容增长工作沉淀为可复用的流程、Prompt、清单和规则。

它不是单纯的文章生成器。文章只是中间产物，真正要解决的是内容增长工作里最容易断掉的几个环节：

1. 选题和业务目标脱节。
2. 正文生成后缺少质检和人工确认。
3. 发布完成后 URL 和数据没有回填。
4. 博客、GEO、AI Bot 数据没有进入下周计划。
5. 周报只做总结，没有反哺下一轮执行。

## V3 核心变化

V3 的重点是页面职责收敛和执行闭环。

| 模块 | V3 职责 |
| --- | --- |
| 首页 | 展示执行队列、Pipeline 状态和当前阻塞项 |
| 周计划 | 只做选题级计划生成、预览和确认 |
| 今日发布 | 批量生成正文、确认已发布、回填正式 URL |
| 草稿预览 | 编辑草稿、查看质检结果、执行 AI 二次质检，通过后复制正文 |
| 数据回传 | 导入或手动补录渠道阅读、点赞、线索等指标 |
| 博客监控 | 诊断官网博客 SEO/GEO 准备度，并生成候选主题 |
| GEO 测试 | 按平台、Prompt 组、蒸馏词和引用层级诊断 AI 搜索可见性 |
| 知识库 | 管理内容来源、规则切片、Chunk 预览和自动抓取设置 |
| 周报复盘 | 汇总发布完成率、数据回传率、GEO 命中率、渠道表现和下周建议 |

## 主流程

1. 在 `周计划` 页面生成并确认本周选题。
2. 在 `今日发布` 页面选择已确认任务，批量生成正文。
3. 进入 `草稿预览` 页面编辑内容，检查质检结果，并执行 AI 二次质检。
4. 人工复制终稿，发布到外部渠道。
5. 回到 `今日发布` 页面确认已发布，并回填正式 URL。
6. 在 `数据回传` 页面导入或补录渠道指标。
7. 在 `博客监控` 和 `GEO 测试` 页面定位官网内容、AI 搜索和引用链路的问题。
8. 在 `周报复盘` 页面复盘本周表现，并形成下一周计划信号。

## 本地启动

安装依赖：

```powershell
npm install
```

启动本地开发服务：

```powershell
npm run dev -- --hostname 127.0.0.1 --port 3047
```

打开：

```text
http://127.0.0.1:3047
```

当前技术栈：

- Next.js 14
- React 18
- Ant Design
- 本地 JSON 状态，用于当前 Demo 和本地试运行
- MySQL 脚本预留，用于后续生产级持久化

## GitHub 文档上传规则

当前仓库只允许上传以下文档：

1. `docs/usage.md`
2. `docs/方案与规划/*.md`

其他内部阶段状态、开发过程、runbook、复盘记录和临时说明文档默认只保留在本地，不上传到 GitHub。

## 不应提交的文件

以下文件不应提交到 GitHub：

- `.env`
- `.env.local`
- `data/workbench-state.json`
- `.next/`
- `node_modules/`
- 日志文件
- 构建缓存

不要提交 API Key、Token、私有链接、真实客户数据或其他敏感信息。
